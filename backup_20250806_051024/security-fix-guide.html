修复 SECURITY DEFINER 视图安全问题

⚠️安全警告

Supabase Security Advisor 检测到安全问题：

  • public.user_stats_view 使用了 SECURITY DEFINER 属性
  • public.draw_stats_view 使用了 SECURITY DEFINER 属性

风险等级： ERROR

🤔 什么是 SECURITY DEFINER 问题？

PostgreSQL 视图默认使用 SECURITY DEFINER 模式，这意味着视图会使用创建者的权限执行， 而不是调用者的权限。这可能导致权限提升和数据泄露风险。

安全模式 权限来源 RLS 策略 安全风险 推荐度
SECURITY DEFINER
(默认)		 视图创建者 可能被绕过 高风险 ❌ 不推荐
SECURITY INVOKER
(推荐)		 视图调用者 严格执行 低风险 ✅ 推荐

🔧 设置操作测试错误修复

解决 "Cannot read properties of undefined (reading 'content')" 错误

❌ 错误信息

Cannot read properties of undefined (reading 'content')

错误位置：system-test.html - 设置操作测试

错误原因：尝试访问不存在的设置项的 content 属性

🔍 问题原因分析

  • 1.
    设置项缺失：数据库中可能没有 'app_name' 设置项
  • 2.
    数据库未初始化：可能没有执行完整的数据库设置脚本
  • 3.
    数据结构问题：设置项存在但 content 字段为空或未定义
  • 4.
    代码缺陷：前端代码没有正确处理空数据的情况

✅ 解决方案

  1. 复制修复脚本

    点击下方按钮复制 SQL 修复脚本

  2. 打开 Supabase SQL Editor

    登录 Supabase 控制台，进入 SQL Editor

  3. 执行修复脚本

    粘贴并执行 SQL 脚本，确保所有设置项存在

  4. 验证修复结果

    重新运行 system-test.html 中的设置操作测试

-- =====================================================
-- 修复设置操作测试错误
-- =====================================================

-- 1. 确保 app_name 设置存在
INSERT INTO settings (key, content, description, category) VALUES
('app_name', '1602 幸运轮盘', '应用名称', 'general')
ON CONFLICT (key) DO UPDATE SET
    content = EXCLUDED.content,
    description = EXCLUDED.description,
    category = EXCLUDED.category,
    updated_at = NOW();

-- 2. 确保其他必需设置存在
INSERT INTO settings (key, content, description, category) VALUES
('app_version', '1.0.0', '应用版本', 'general'),
('max_draw_chances', '1', '每个用户最大抽奖次数', 'game'),
('announcement', '欢迎参与1602幸运轮盘活动！', '公告内容', 'content'),
('contact_info', 'support@1602.com', '联系信息', 'general'),
('is_active', 'true', '活动是否激活', 'game')
ON CONFLICT (key) DO UPDATE SET
    content = EXCLUDED.content,
    updated_at = NOW();

-- 3. 验证设置项
SELECT 
    '✅ 设置项验证结果：' as 验证结果;

SELECT 
    key as 设置键,
    content as 内容,
    CASE 
        WHEN content IS NOT NULL AND content != '' THEN '✅ 正常'
        ELSE '❌ 异常'
    END as 状态
FROM settings 
WHERE key IN ('app_name', 'app_version', 'max_draw_chances', 'announcement', 'contact_info', 'is_active')
ORDER BY key;

🏥 系统健康检查修复指南

解决 "data.forEach is not a function" 错误

🚨 错误信息

错误: 系统健康检查失败: data.forEach is not a function

原因: 系统健康检查函数返回的数据格式不是数组，导致前端无法使用 forEach 方法遍历数据。

🔍 问题分析

📊 数据格式问题

system_health_check() 函数可能返回了非数组格式的数据，如对象、字符串或 null。

🔧 函数定义问题

数据库中的健康检查函数可能存在定义错误或返回类型不正确。

🌐 RPC 调用问题

Supabase RPC 调用可能没有正确处理函数的返回值。

⚡ 前端处理缺陷

前端代码没有对返回数据进行类型检查就直接调用 forEach。

🛠️ 解决方案

修复步骤

1
复制修复脚本
点击下方的"复制脚本"按钮，复制完整的修复脚本到剪贴板。
2
打开 Supabase SQL Editor
登录 Supabase 控制台，进入项目的 SQL Editor 页面。
3
执行修复脚本
将复制的脚本粘贴到 SQL Editor 中，点击"Run"执行脚本。
4
验证修复结果
返回系统测试页面，重新运行系统健康检查，确认问题已解决。

📝 修复脚本

fix-system-health-check.sql
-- ===================================================== -- 修复系统健康检查函数 -- ===================================================== -- 删除现有的系统健康检查函数 DROP FUNCTION IF EXISTS system_health_check(); -- 重新创建系统健康检查函数 CREATE OR REPLACE FUNCTION system_health_check() RETURNS TABLE( check_name TEXT, status TEXT, details TEXT ) AS $$ BEGIN -- 检查用户表 RETURN QUERY SELECT 'users_table'::TEXT as check_name, CASE WHEN (SELECT COUNT(*) FROM users) > 0 THEN 'OK' ELSE 'WARNING' END::TEXT as status, ('Total users: ' || (SELECT COUNT(*) FROM users))::TEXT as details; -- 检查抽奖历史表 RETURN QUERY SELECT 'draw_history_table'::TEXT as check_name, CASE WHEN (SELECT COUNT(*) FROM draw_history) > 0 THEN 'OK' ELSE 'INFO' END::TEXT as status, ('Total draws: ' || (SELECT COUNT(*) FROM draw_history))::TEXT as details; -- 检查设置表 RETURN QUERY SELECT 'settings_table'::TEXT as check_name, CASE WHEN (SELECT COUNT(*) FROM settings) > 0 THEN 'OK' ELSE 'WARNING' END::TEXT as status, ('Total settings: ' || (SELECT COUNT(*) FROM settings))::TEXT as details; -- 检查数据库连接 RETURN QUERY SELECT 'database_connection'::TEXT as check_name, 'OK'::TEXT as status, ('Connected at: ' || NOW()::TEXT)::TEXT as details; END; $$ LANGUAGE plpgsql; -- 测试系统健康检查函数 SELECT * FROM system_health_check();
